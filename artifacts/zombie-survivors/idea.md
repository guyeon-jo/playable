# Zombie Survivors — 5분 아포칼립스

## Problem Statement

HMW — 브라우저 탭 하나에서, 5분 안에 "내가 무적이 됐다"는 파워 판타지를 완결시킬 수 있을까?

## Recommended Direction

**미니멀 5분 완결 루프** — VS의 본질은 스노우볼 쾌감이다. 무기 진화 시스템이나 메타 진행이 아니라, "레벨업 선택 → 점점 강해짐 → 화면을 뒤덮는 좀비 떼를 밀어내는 순간"이 핵심이다.

웹에서 설치 없이 즉시 플레이 가능하다는 것이 실질적 차별점이다. Next.js 위에서 Canvas API로 구현하면 외부 게임 엔진 의존 없이 유지·확장이 쉽다.

5분 타이머가 명확한 종료 조건을 만들어 "한 판"의 완결감을 보장한다. 좀비 아포칼립스 테마는 적의 수가 많아질수록 자연스럽게 느껴지고, 시각적 밀도가 쾌감으로 직결된다.

## Key Assumptions to Validate

- [ ] Next.js에서 Canvas requestAnimationFrame 루프가 60fps 안정적으로 동작한다 — 빈 캔버스에서 200개 오브젝트 렌더링 프로파일링으로 검증
- [ ] 3-5종 무기만으로도 레벨업 선택이 매 런마다 다르게 느껴진다 — 직접 플레이 5회로 검증
- [ ] 5분이라는 시간이 "짧지만 달리고 싶다"는 감각을 준다 — 내부 플레이테스트로 검증

## MVP Scope

**포함:**
- Canvas 기반 게임 루프 (requestAnimationFrame)
- 캐릭터 2종, WASD / 방향키 이동
  - **총잡이** (기본 해금): 원거리 자동 공격, 이동속도 보통
  - **검사** (첫 클리어 또는 게임오버 후 해금): 근접 범위 자동 공격, 이동속도 빠름·체력 높음
- 자동 공격 (가장 가까운 적 타겟)
- 적 3종: 일반 좀비(느림), 달리는 좀비(빠름), 탱커(느리고 단단함)
- 무기/스킬 풀 — 캐릭터별로 다른 선택지 등장
  - 총잡이: 권총, 산탄총, 화염방사기, 수류탄, 전기 충격
  - 검사: 검 회전, 돌진 베기, 충격파, 흡혈 베기, 방패 강타
- 레벨업 시 3개 중 1개 선택
- 경험치 드롭 → 수집 → 레벨업
- 5분 타이머 → 생존 시 클리어 화면
- 죽으면 게임오버 화면 + 재시작
- **캐릭터 선택 화면**: 최초엔 총잡이만 활성, 한 판 완료(클리어 또는 게임오버) 후 검사 해금

**빠짐:**
- 3종 이상 캐릭터 추가
- 무기 진화 시스템
- 모바일/터치 지원
- 사운드/BGM
- 리더보드/점수 저장

## Not Doing (and Why)

- **무기 진화 시스템** — 조합 로직이 무기 밸런싱 복잡도를 10배 올린다. 핵심 쾌감은 진화 없이도 달성 가능.
- **3종 이상 캐릭터** — 검사 해금 한 개가 "또 하고 싶다"는 동기를 만들기엔 충분. 그 이상은 밸런싱 부채가 기하급수적으로 늘어난다.
- **멀티플레이** — 동기화 복잡도가 게임 자체보다 크다.
- **모바일 지원** — 키보드 이동이 핵심 UX. 가상 조이스틱은 별도 feature로.
- **사운드** — 비주얼 임팩트가 먼저. 사운드는 post-MVP.
